What Students Find Hard

  • Thinking about the world as snapshots (states) instead of step-by-step instructions
  • Remembering all the rules for when actions are allowed
  • Understanding why the computer cannot find a plan
  • Finding mistakes in Prolog code when things go wrong

Helpful Tips

1

Start Small

Use just 2 places instead of 5, or 2 objects instead of 10.

This makes it easier to understand and test your code. Once it works, add more complexity later.

2

Draw Pictures

Draw each situation on paper with arrows showing what actions change.

Visualizing the states helps you see what needs to be true before an action and what changes after.

3

Write Things Down First

List all objects, places, and facts before writing code.

Make a simple table: Objects | Locations | Facts | Actions. This helps you organize your thoughts.

4

Think Step by Step

For each action, ask: What must be true before? What changes after? What stays the same?

If you can explain your action in simple words like a story, it will be easier to write the Prolog code correctly.

5

Test One Thing at a Time

Test single actions first, like checking if the monkey can move, before testing the whole plan.

Check the starting situation → test if actions are allowed → see what changes. This helps you find mistakes quickly.

6

Use Good Names

Write at(robot, l1) instead of short names like a(r, 1).

Clear names make your code easier to read and understand later. Good names help you remember what everything means.

Common Mistakes

Forgetting "NOT" Rules

Some actions need something to be NOT true.

  • Example: The monkey can only move if it is not on the box
  • Problem: Many students forget to check negative conditions
  • Result: The planner fails or finds wrong plans
  • Solution: Always check both what must be true AND what must be false
